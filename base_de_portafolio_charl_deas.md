# Charl!deas — Documento Base de Portafolio

Este documento define **la estructura, narrativa, textos base y criterios** para construir el portafolio de Charl!deas. Está pensado como **fuente única de verdad** para luego llevar a diseño (Illustrator) y ejecución (Cursor / código).

---

## 1. Objetivo del portafolio

**Objetivo principal:**
Mostrar capacidad real para **diseñar y construir productos digitales funcionales** (no demos), combinando:
- pensamiento de producto
- experiencia de usuario
- criterio técnico
- ejecución completa

**Objetivo secundario:**
Servir como base comercial para Charl!deas (clientes, proyectos a medida, branding).

---

## 2. Público objetivo

El portafolio debe hablarle principalmente a:
- dueños de negocios
- pymes
- profesionales independientes
- startups pequeñas
- personas que necesitan *soluciones*, no buzzwords

NO está orientado a:
- recruiters ultra técnicos
- empresas enterprise
- evaluaciones algorítmicas

El lenguaje debe ser **claro, humano y directo**.

---

## 3. Posicionamiento

**Qué es Charl!deas:**

> Charl!deas es un estudio de productos digitales que crea sistemas web y marcas visuales para ayudar a personas y negocios a organizarse, vender y trabajar mejor.

**Qué NO es Charl!deas:**
- una fábrica de landing pages genéricas
- una agencia inflada
- un showcase técnico vacío

---

## 4. Identidad visual (logo y colores)

### Logo

El logo de **Charl!deas** debe utilizarse siempre como elemento de identificación principal.

**Reglas de uso:**
- Preferir el logo en su versión original (SVG inline cuando sea posible).
- Mantener suficiente espacio en blanco alrededor del logo.
- Evitar efectos innecesarios (sombras duras, outlines, distorsiones).
- El logo debe sentirse **limpio, moderno y confiable**, no decorativo.

### Colores principales

**Color base del logo (principal):**
- Celeste: `#00B4B9`

Este color debe ser el eje visual de la web.

**Uso recomendado:**
- CTAs principales
- Elementos activos
- Highlights
- Detalles de identidad

### Degradados (opcional)

Si se utilizan degradados, deben construirse **siempre a partir del celeste del logo**, combinándolo con:
- Azul más oscuro (ej: `#0A4F6E` o similar)
- Verde más profundo (ej: `#0E7C6B` o similar)

Nunca usar degradados estridentes o con colores que rompan la identidad.

### Paleta de apoyo

- Fondos claros: blancos rotos o grises muy suaves
- Fondos oscuros (si aplica): azules o verdes muy oscuros, no negro puro
- Texto principal: gris muy oscuro / casi negro

La identidad debe transmitir:
> claridad · calma · confianza · utilidad

## 6. Proyectos — Orden y rol

### 6.1 SushiWey 🍣 (Flagship)

**Rol:** Proyecto principal

**Descripción ampliada:**

SushiWey es un sistema web desarrollado a medida para un local de sushi real. Permite a los clientes realizar pedidos en línea y seguir su estado en tiempo real, mientras el negocio gestiona órdenes, clientes, direcciones, métricas y operación diaria desde un panel administrativo.

Incluye flujos completos de compra, tracking de pedidos, generación de comandas, geolocalización y un dashboard operativo pensado para uso real, incluso desde dispositivos móviles.

**Mensaje clave:**
> Sistema comercial real, usado en operación diaria.

---

### 6.2 Lukita 💰

**Rol:** Producto personal sólido

**Descripción ampliada:**

Lukita es una aplicación de finanzas personales creada para el control diario del dinero. Permite gestionar cuentas, ingresos, gastos, presupuestos y movimientos recurrentes, con soporte offline-first y sincronización automática.

El foco del proyecto está en la simplicidad, el uso real y la flexibilidad mensual, adaptándose a la forma en que las personas realmente administran su dinero.

**Mensaje clave:**
> Aplicación de finanzas personales pensada para uso diario, offline-first.

---

### 6.3 Postly 🗂️

**Rol:** Proyecto de UX y foco

**Descripción ampliada:**

Postly es una aplicación de gestión de tareas visual basada en tableros y post-its. Está diseñada para uso personal, priorizando la claridad mental, la rapidez y la mínima fricción.

El proyecto se centra en decisiones de experiencia de usuario: drag & drop fluido, edición directa, uso en móvil y una interfaz limpia que evita la sobrecarga de opciones.

**Mensaje clave:**
> Herramienta simple de gestión de tareas, diseñada para claridad mental.

---

### 6.4 NutriTrack 🥗

**Rol:** Proyecto secundario / experimental

**Descripción ampliada:**

NutriTrack es una plataforma de seguimiento nutricional enfocada en hábitos diarios. Permite registrar comidas, macros, hidratación y actividad física, entregando visualizaciones claras y cálculos automáticos.

El proyecto explora el uso de datos, visualización y apoyo automatizado para mejorar la conciencia alimentaria y la constancia.

**Mensaje clave:**
> Exploración de hábitos, datos y seguimiento nutricional.

---

### 6.5 Sloty 📅

**Rol:** Work in Progress

**Descripción ampliada:**

Sloty es una plataforma web para la gestión de citas y horarios profesionales. Permite a prestadores de servicios configurar disponibilidad, servicios, reglas de agenda y recibir reservas desde un perfil público.

El foco del proyecto está en la lógica de disponibilidad, reglas de agenda y flujos claros entre cliente y profesional. Actualmente se encuentra en desarrollo activo.

**Mensaje clave:**
> Plataforma de gestión de citas en desarrollo, enfocada en reglas de agenda y disponibilidad.

## 6. Página de proyecto — Plantilla

Todas las páginas de proyecto deben seguir esta estructura:

1. **Contexto**
   - Por qué nació el proyecto

2. **Problema**
   - Qué necesidad real existía

3. **Solución**
   - Qué se construyó

4. **Qué hace el sistema**
   - Funcionalidades clave (lenguaje humano)

5. **Decisiones clave**
   - UX
   - alcance
   - reglas

6. **Resultado**
   - uso real / estado actual

7. **Aprendizajes**
   - qué funcionó
   - qué mejorarías

8. **Stack (breve)**
   - solo como soporte, no protagonista

---

## 7. Herramientas y proceso

En Charl!deas utilizamos las herramientas que mejor se ajustan a cada proyecto para trabajar de forma eficiente y ordenada.

La tecnología y las herramientas son un medio, no el objetivo. El foco siempre está en:
- entender el problema
- diseñar una buena experiencia
- construir soluciones mantenibles
